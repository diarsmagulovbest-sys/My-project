Начинаем новую функцию: Adaptive Mentor Profiles.

Текущая AI-интеграция:

* в приложении уже подключён Gemini 2.5 Flash;
* AI-запросы идут через Supabase Edge Function;
* GEMINI_API_KEY хранится только в Supabase Secrets;
* frontend не должен видеть AI-ключ;
* не менять AI-провайдера;
* не подключать OpenAI, Claude или другую модель;
* не переносить AI-вызовы во frontend;
* все mentor profiles должны работать через текущую Gemini 2.5 Flash интеграцию.

Идея функции:
Приложение должно автоматически подбирать стиль AI-наставника под цель пользователя.

Примеры:

* цель про кубик Рубика → наставник по головоломкам / Rubik's Cube coach;
* цель про тхэквондо → безопасный спортивный наставник;
* цель про программирование → coding mentor;
* цель про английский → language learning mentor;
* цель про экзамены → study/exam mentor;
* если категория непонятна → general goal mentor.

Важно:
Это не разные AI-модели. Это разные system prompts / mentor profiles для одной текущей AI-интеграции через Gemini 2.5 Flash.

Правила для mentor profiles:

1. Каждый профиль должен менять стиль наставника, но не ломать общую логику приложения.
2. Профиль должен использоваться при:

   * генерации уточняющих вопросов;
   * генерации дорожной карты;
   * AI Mentor chat.
3. Профиль не должен заставлять AI обещать гарантированный результат.
4. Профиль не должен давать опасные инструкции.
5. Для спорта, фитнеса, боевых искусств, здоровья и похожих целей нужны осторожные safety rules.
6. Если цель опасная, медицинская, травмоопасная или требует взрослого специалиста, AI должен отвечать осторожно и советовать обратиться к квалифицированному человеку.
7. Если категория непонятна, использовать general.
8. Если confidence классификации низкий, использовать general.
9. Не добавлять отдельную админку для профилей.
10. Не хранить все профили в базе данных на первой версии.

Где хранить mentor profiles:

* сначала предложи хранить их в TypeScript-константе;
* не создавай отдельные таблицы для mentor profiles, пока функция не стабилизируется;
* структура должна быть простой и расширяемой.

Предложи пример TypeScript-структуры, например:

Как определять категорию:

* сначала предложи простую AI-классификацию цели через текущую Gemini Edge Function;
* классификация должна возвращать строго ограниченный список категорий;
* ответ классификации должен проверяться через Zod;
* если результат невалидный, использовать general;
* если confidence низкий, использовать general;
* не делать сложную ML-систему;
* не добавлять embeddings или vector database на этой версии.
